工厂方法模式:
    定义了一个创建对象的接口,但是由子类决定要实例化的类是那一个。工厂方法让类把实例推迟到子类,
    主要体现在编写创建者类时，不需要实际创建的产品是那一个,选择使用了哪个子类,就决定了实际创建
    的产品是什么。
角色分析:
    Product:产品接口
    ConcreteProduct:具体的产品
    Creator:产品创造者接口
    ConcreteCreator:具体的创造者
优点:
    当只有一个ConcreteCreator的时候,工厂方法模式尤什么有点?
    尽管只有一个具体的创建者,工厂方法模式依然很有用,它帮助我们将产品的"实现"从"使用"中解耦，
    如果增加产品护着改变产品的实现,Creator并不会受到影响,因为Creator与任何ConcreProduct之间都
    不是紧耦合
深入分析:
    简单工厂和工厂方法的区别?
    简单工厂把全部的事情,在一个地方都处理完了,然而工厂方法确实创建一个框架,让子类决定如何实现
