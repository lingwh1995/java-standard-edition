抽象工厂模式:
    提供一个创建一系列相关或者相互依赖对象的接口,而无需指定他们具体的类
    用来创建一系列的产品对象,而且这一些列对象是构建新对象所需要的重要组成部分,
    也就是这一系列被创建的对象之间是要有相互约束的
角色分析:
    AbstractFactory:
        抽象工厂,定义创建一系列产品对象的操作接口
    ConcreteFactory:
        具体的工厂,实现抽象工厂定义的方法,具体实现一系列产品对象的创建
    AbstractProduct:
        定义一类产品对象的接口
    ConcreteProduct:
        具体的产品实现对象,通产在具体的工厂里面，会选择具体的产品实现对象,来
        创建符合抽象工厂定义的方法返回的产品类型的对象
    Client:
        客户端,主要使用抽象工厂来获取一系列所需要的产品对象,然后面向这些产品对象
        的接口编程,以实现主要功能
