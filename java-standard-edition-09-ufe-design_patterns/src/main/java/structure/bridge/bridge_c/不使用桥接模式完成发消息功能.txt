短信发送组件设计需求:
    可以使用邮件和手机短信两种方法发送消息
    消息的类型有:
        普通消息:直接发送就行
        加急消息:发送的时候在内容中添加“加急消息”这四个字，另外加急消息会提供监控的方法，让客户端可以随时通过这个方法来了
                解对于加急消息的处理进度。比如，相应的人员是否接收到这个信息，相应的处理工作是否已经展开。因此加急消息需要扩展出一个新的接口，除了基本的发送消息的功能，还需要添加监控功能。
        特急消息:在发送消息后，没有监控功能，没有完成就催促，所以要添加催促功能
存在问题:
    1.如果新增加一种消息的发送类型,如:十万火急消息
        需要再增加两个实现类，分别是:十万火急消息邮件发送类  十万火急消息短信发送类
    2.如果要新增加一个发送方式，如:使用QQ发送
        以目前有三种消息类型为例，需要增加:
            QQ发送普通消息类
            QQ发送加急消息类
            QQ发送特急消息类
导致问题:不管是问题1还是问题2，都会导致类爆炸
