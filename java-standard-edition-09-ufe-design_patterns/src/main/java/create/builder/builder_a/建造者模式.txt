建造者模式:
	场景:我们需要建造一个复杂的产品，比如神州飞船，这个复杂产品的创建，有一个问题需要处理 ，就是装配子组件需要多个步骤，
		在实际开发中，我们所需要的对象构建时，也非常复杂，需要多个步骤
	本质:分离对象子组件的单独构造(由Builder来负责)和装配(由Director负责)，从而构造出许多复杂对象，这种模式适用于
		某个对象的构建过程复杂的情况下使用
		由于实现了构建器和装配器解耦，不同的构建器，相同的装配器，可以做出不同的对象，不同的装配置，相同的构建器，也可
		以做出不同的对象，不同的装配顺序可以做出不同的对象，实现了构建算法集和装配算法的解耦使用，实现了更好的复用
